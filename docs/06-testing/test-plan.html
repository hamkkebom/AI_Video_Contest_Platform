🧪 테스트 계획서

Phase 1 릴리스 기준으로 기능 안정성과 운영 리스크를 검증하기 위한 테스트 전략 문서

문서 정보: v0.1-mockup | 최종 수정: 2026-02-19 | 상태: 목업 기준 초안

1. 테스트 목표

핵심 기능(F-001~F-006)의 제출-심사-결과 흐름을 안정적으로 보장하고, 상태값 전이(ST-001, ST-003, ST-004)와 권한 경계(401/403)를 포함한 리스크 구간을 사전에 검증한다.

2. 테스트 전략

구분 목적 주요 범위 도구
Unit 도메인 로직/유틸 단위 정확성 검증 입력 검증, 상태 전이 계산, 점수 집계 로직 Vitest
Integration API-DB-권한 흐름 통합 검증 API-004, API-005, API-007, API-013~API-015 Vitest + mock API harness
E2E 역할별 사용자 여정과 회귀 검증 참가자 제출, 심사자 평가, 주최자 결과 발표, 관리자 접근 제어 Playwright

3. 커버리지 목표

대상 목표 기준
핵심 경로 80% 이상 F-001, F-002, F-003, F-006 관련 경로 기준
예외/오류 경로 70% 이상 400/401/403/404/409 응답 및 마감 처리 기준
역할 접근 경로 100% 점검 guest/participant/host/judge/admin(ST-001)별 허용/차단 검증

4. 테스트 환경

환경 용도 검증 항목
Local 개발 중 빠른 피드백 Unit/Integration 기본 회귀, 입력 검증, 에러 처리
Staging 릴리스 전 통합 검증 E2E 전체 시나리오, 마감 시점 부하 리허설, 권한 점검
Production 배포 후 안정성 확인 스모크 테스트, 모니터링 알람, 핵심 API 상태 확인

5. 테스트 범위

  • 기능 테스트: F-001~F-006 핵심 흐름과 F-004/F-005 사용자 반응 기능
  • 예외 테스트: 잘못된 파일 형식, 2GB 초과 파일, 중복 좋아요(API-007 409), 미인증 요청(401)
  • 마감 시점 테스트: ST-003 open→closed 전이 시점 제출 허용/차단, 동시 요청 안정성
  • 역할별 접근 테스트: ST-001 역할 기반 화면/엔드포인트 접근 통제(403 포함)
  • 결과 무결성 테스트: API-014 점수, API-015 집계/정렬 일관성 검증

6. 결함 우선순위 및 종료 기준

심각도 정의 릴리스 기준
Critical핵심 흐름 중단, 데이터 손상, 권한 우회0건 필수
High주요 기능 실패, 마감/심사 지연허용 기준 이내 및 대응 계획 필수
Medium우회 가능한 기능 저하차기 스프린트 이관 가능

주차 중점 활동 출력물

QA 2주차 E2E 전체 시나리오, 마감 시점/역할 권한 집중 점검 결함 리포트 v2, 회귀 목록
