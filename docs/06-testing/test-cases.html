5. 추적 기준

  • Critical 케이스(TC-001, 002, 003, 005, 010, 011, 015, 017, 020, 022)는 배포 전 100% 재실행한다.
  • 모든 테스트 결과는 기능 ID(F-XXX), API ID(API-XXX), 상태값 ID(ST-XXX)와 함께 기록한다.
  • 결함 수정 후에는 원인 케이스와 인접 경로를 묶어 회귀 테스트를 수행한다.
