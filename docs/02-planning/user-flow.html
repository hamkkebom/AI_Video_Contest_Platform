🔄 사용자 흐름도 (User Flow)

함께봄 AI 영상 공모전 플랫폼 - 역할별 핵심 사용자 흐름 정의

문서 정보: v0.1-mockup | 최종 수정: 2026-02-19 | 상태: 목업 기준 초안
흐름 기준: 5개 역할 × 핵심 시나리오 = 6개 플로우

목차

  • 1. 참가자 흐름
  • 2. 주최자 흐름
  • 3. 심사위원 흐름
  • 4. 관리자 흐름
  • 5. 검색/갤러리 흐름
  • 6. 인증/온보딩 흐름

1. 참가자 흐름

참가자(participant)가 플랫폼에 가입하여 공모전을 탐색하고, 영상 작품을 제출한 뒤 결과를 확인하는 전체 흐름입니다.

단계 페이지 행동 관련 기능
1 /signup 개인 계정 생성, 이메일 인증 F-007
2 /contests 공모전 목록 탐색, 필터/검색 F-011
3 /contests/:id 공모전 상세 확인, 참가 조건 확인 F-001
4 /contests/:id 영상 작품 제출 (제목, 설명, 파일) F-001
5 /my/submissions 제출 상태 확인 (submitted → under-review) F-001
6 /contests/:id 심사 결과 확인, 수상 여부 확인 F-003
7 /my/analytics 참여 이력, 수상 내역, 조회수 확인 F-008

2. 주최자 흐름

주최자(host)가 공모전을 생성하고, 접수를 관리하며, 심사를 설정하고, 결과를 발표하는 전체 운영 흐름입니다.

단계 페이지 행동 관련 기능
1 /my/profile 주최 기능 토글 ON (기업/개인) F-007
2 /dashboard/contests/new 신규 공모전 생성 (제목, 설명, 일정, 상금) F-001
3 /dashboard/contests/:id/judges 심사위원 초대 (이메일 기반) F-006
4 /dashboard/contests/:id 공모전 공개 (draft → open) F-001
5 /dashboard/contests/:id/submissions 접수작 확인 및 관리 F-001
6 /dashboard/contests/:id 접수 마감, 심사 시작 (open → closed → judging) F-002
7 /dashboard/contests/:id 결과 발표 (judging → completed) F-003
8 /dashboard/reports 결과 보고서 생성/다운로드 F-003

3. 심사위원 흐름

심사위원(judge)이 초대를 수락하고, 배정된 작품을 채점하며, 심사를 완료하는 흐름입니다. 심사위원 역할은 초대제로만 활성화됩니다.

단계 페이지 행동 관련 기능
1 /invite/:token 이메일 초대 링크 클릭, 초대 수락 F-006
2 /judging 심사 대시보드에서 배정된 공모전 확인 F-002
3 /judging/:contestId 심사 대상 작품 목록 확인 F-002
4 /judging/:contestId/:submissionId 개별 작품 채점 (점수 부여, 코멘트 작성) F-002, F-009
5 /judging/:contestId 다음 작품으로 이동 또는 심사 완료 F-002

4. 관리자 흐름

관리자(admin)가 플랫폼 전체를 운영하는 핵심 흐름입니다. 대시보드에서 회원 관리, 공모전 승인, 통계 분석, 콘텐츠 관리를 수행합니다.

단계 페이지 행동 관련 기능
1 /admin/dashboard 전체 현황 확인 (사용자 수, 공모전 수, 트래픽) —
2 /admin/users 회원 목록 관리 (검색, 필터, 역할 변경) F-007
3 /admin/analytics 통계 분석 (사용자, 트래픽, 매출) F-008
4 /admin/articles 뉴스/아티클 등록 및 편집 —
5 /admin/inquiries 사용자 문의 처리 —
6 /admin/agency-requests 대행 요청 검토 및 승인 F-010

5. 검색/갤러리 흐름

모든 사용자(비회원 포함)가 갤러리를 탐색하고, 작품을 감상하며, 좋아요를 표시하는 흐름입니다. 통합 검색을 통해 공모전, 크리에이터, 갤러리를 한 번에 탐색할 수 있습니다.

단계 페이지 행동 관련 기능
1 / 또는 /search 검색어 입력 또는 갤러리 진입 F-011
2 /gallery 갤러리 탐색, 카테고리/기간/평점 필터 F-004, F-011
3 /gallery 작품 상세 모달 또는 확대 보기 F-004
4 /gallery 좋아요 표시 (로그인 필요) F-005
5 /creators/:id 크리에이터 프로필 방문 —

6. 인증/온보딩 흐름

비회원(guest)이 플랫폼에 가입하고, 계정 유형(개인/기업)을 선택하며, 기능 토글을 설정하는 온보딩 흐름입니다.

단계 페이지 행동 관련 기능
1 / 랜딩 페이지 방문, CTA 클릭 —
2 /signup 계정 유형 선택 (개인/기업), 정보 입력 F-007
3 /login 로그인 (이메일/소셜) F-007
4 /my/profile 프로필 완성, 기능 토글 설정 F-007
5 역할별 대시보드 활성화된 기능에 따라 해당 대시보드 진입 F-007
