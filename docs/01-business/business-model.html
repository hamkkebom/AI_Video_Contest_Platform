수익 모델 개요

Phase 1은 무료 기반 확산과 운영 데이터 축적에 집중하고, 유료는 분석 고도화/운영 효율 기능을 중심으로 단계적으로 전환합니다. 과금 대상은 주최자의 운영 고급 기능, 참가자의 분석 기능, 그리고 Phase 2 이후 대행 연결 수수료입니다.

역할별 요금제 (PricingPlan 기반)

PricingPlan(id, role, name, monthlyPrice, yearlyPrice, active, featureKeys) 타입을 기준으로 목업용 요금제를 정의합니다.

Plan ID Role Plan Name 월간 요금 (예시) 연간 요금 (예시) 핵심 포함 기능
plan-free-participant participant 무료 무과금 무과금 work-performance(기본), 제출/갤러리 참여
plan-standard-host host 스탠다드 월 X만원대 연 X만원대 (할인) submission-status, participant-distribution, channel-performance

수수료 구조

핵심 수익은 구독 + 운영 수수료로 구성하고, 수수료는 전환 지표가 안정화되는 Phase 2부터 적용합니다.

구분 적용 시점 대상 정책 방향
공모전 주최 수수료 Phase 1 후반~2 기업 주최자 공모전 생성/운영 패키지별 정률 또는 정액 모델(확정 전)
대행 연결 수수료 Phase 2 의뢰 성사 건 공모전-대행 연결 성과 기반 수수료(예시 비율, 추후 확정)

유입 전략 (UTM 전환추적 기반)

채널별 UTM 파라미터를 first-touch/last-touch로 수집하고, 가입-참여-제출-의뢰 전환까지 연결합니다.

채널 UTM 예시 핵심 목적 주요 측정 지표
콘텐츠/SEO utm_source=organic 참가자 유입 확대 가입 전환율, 갤러리 체류시간
퍼포먼스 광고 utm_source=paid&utm_campaign=contest_launch 공모전 모집 속도 확보 CPA, 제출 완료율
파트너/커뮤니티 utm_source=partner 신뢰 기반 주최자 확보 주최자 문의수, 개설 전환율

외부 대행 서비스 연결 전략

공모전 플랫폼과 대행 플랫폼은 별도 운영을 유지하되, 내부 API를 통해 메타데이터만 연동합니다. 영상은 Mux ID 참조 방식으로 재활용하고 파일 중복 저장은 하지 않습니다. 사용자는 공모전에서 대행 CTA를 통해 이동하고, 의뢰 성사 데이터는 Phase 2에 수익 분석으로 연결합니다.

flowchart LR IN[유입 채널 UTM] --> JOIN[회원가입/참가] JOIN --> CONTEST[공모전 참여] CONTEST --> SUB[영상 제출/심사] SUB --> GAL[갤러리 노출] GAL --> CTA[대행 연결 CTA] CTA --> AGENCY[대행 플랫폼 의뢰] AGENCY --> REV1[구독 매출] AGENCY --> REV2[Phase 2 연결 수수료]

기능별 과금 매트릭스 (DEFAULT_FEATURE_ACCESS 기반)

현재 기본 접근 정책은 free 여부를 기준으로 하며, 스탠다드/프리미엄은 free=false 기능을 단계적으로 해제하는 구조입니다.
