1. 리스크 평가 기준

확률(높음/중간/낮음)과 영향도(높음/중간/낮음)를 결합해 우선순위를 판단하며, 상태값 정의 문서의 ST-003(ContestStatus), ST-004(SubmissionStatus) 전이 실패 가능성을 운영 리스크 평가에 반영한다.

R-007 비즈니스 저작권 분쟁 발생 시 작품 비공개/삭제 대응 지연 높음 높음 신고 접수 SLA, 임시 블라인드 상태(ST-004 연계 정책) 정의, 법무 협업 루틴 구축 운영
R-008 운영 동시 심사자 100명 환경에서 점수 저장(API-014) 충돌 위험 중간 높음 낙관적 잠금/중복 제출 차단, 저장 큐 분리, 심사 세션 타임아웃 정책 백엔드
R-009 운영 마감 시점 직전 문의 폭주로 사용자 응답 지연 중간 중간 FAQ 우선 노출(API-009), 자동 응답 템플릿, 긴급 티켓 분류 운영 QA

4. 운영 원칙

  • Phase 1 종료 전 R-001, R-002, R-007은 반드시 리허설 기반으로 재평가한다.
  • 모든 High/Critical 리스크는 테스트 계획서의 마감 시점 테스트와 연결해 검증한다.
  • 역할 접근 문제는 ST-001(UserRole)과 API 권한 규칙(401/403) 기준으로 대응한다.
